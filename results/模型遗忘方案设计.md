# 大模型选择性遗忘与能力增强方案

## 1. 核心思想

传统剪枝只考虑"层重要性"，但忽略了知识类型的差异。本方案提出**知识分类遗忘**机制：

| 知识类型 | 特征 | 处理策略 |
|---------|------|---------|
| **核心能力** | 推理、逻辑、语言理解、模式识别 | 保留并强化 |
| **细节记忆** | 事实知识、训练数据片段、特定表达 | 选择性遗忘 |
| **冗余参数** | 对输出影响极小的权重 | 直接剪枝 |

## 2. 三阶段遗忘流程

```
阶段1: 知识分类 → 阶段2: 选择性遗忘 → 阶段3: 能力蒸馏
    ↓                    ↓                    ↓
识别核心vs细节      遗忘细节+剪枝冗余      蒸馏强化核心能力
```

### 阶段1：知识分类（Knowledge Classification）

**目标**：识别模型中哪些参数负责"核心能力"，哪些负责"细节记忆"

**方法**：
1. **任务敏感度分析**：用核心能力测试集 vs 事实记忆测试集分别计算梯度
2. **参数重叠度**：计算两类任务对同一参数的依赖程度
3. **分类标签**：给每个参数打标签 - `core` / `memory` / `neutral`

```python
# 核心能力测试集示例
CORE_TASKS = [
    "logical_reasoning",    # 逻辑推理
    "math_problem",         # 数学计算
    "code_generation",      # 代码生成
    "text_summarization",   # 文本摘要
    "instruction_following" # 指令遵循
]

# 细节记忆测试集示例
MEMORY_TASKS = [
    "fact_recall",          # 事实回忆
    "data_reproduction",    # 数据复现
    "exact_quote",          # 精确引用
    "template_matching"     # 模板匹配
]
```

### 阶段2：选择性遗忘（Selective Forgetting）

**目标**：有选择地"擦除"细节记忆，同时保护核心能力

**方法**：
1. **梯度上升遗忘**：对记忆类参数执行梯度上升（反向训练），主动"忘记"
2. **噪声注入**：对记忆类参数添加高斯噪声，破坏精确记忆
3. **保护性微调**：对核心类参数用核心任务数据微调保护

```python
# 遗忘损失函数
def forgetting_loss(model, memory_data, core_data, param_tags):
    # 对记忆数据：最大化损失（遗忘）
    loss_memory = -model(memory_data).loss  # 梯度上升
    
    # 对核心数据：最小化损失（保护）
    loss_core = model(core_data).loss       # 正常训练
    
    # 根据参数标签加权
    loss = α * loss_memory + β * loss_core
    return loss
```

### 阶段3：能力蒸馏（Capability Distillation）

**目标**：在压缩后的模型上强化核心能力

**方法**：
1. **原始模型作为教师**：用完整模型生成高质量输出
2. **蒸馏核心任务**：重点蒸馏推理、逻辑等核心任务
3. **渐进式蒸馏**：从简单到复杂，逐步提升能力

## 4. 关键技术创新

### 4.1 参数级知识标签

```python
# 每个参数的知识标签
param_tags = {
    "layer.0.self_attn.q_proj.weight": "core",      # 核心
    "layer.5.mlp.up_proj.weight": "memory",         # 记忆
    "layer.10.self_attn.o_proj.weight": "neutral",  # 中性
    ...
}
```

### 4.2 遗忘强度控制

```python
# 可配置遗忘强度
FORGETTING_STRENGTH = {
    "light": 0.2,    # 轻度遗忘，性能损失<2%
    "medium": 0.5,   # 中度遗忘，性能损失5-10%
    "heavy": 0.8,    # 重度遗忘，性能损失15-20%
}
```

### 4.3 混合压缩策略

```
最终模型 = 剪枝(层移除) + 遗忘(参数擦除) + 量化(精度降低)
```

## 7. 与现有剪枝的关系

本方案是现有剪枝流程的**增强版**：

```
现有流程: 重要性评估 → 层剪枝 → 微调恢复
新流程:   知识分类 → 选择性遗忘 → 层剪枝 → 能力蒸馏 → 量化
```

新流程在剪枝前后各增加了一个阶段，实现更精细的知识管理。
