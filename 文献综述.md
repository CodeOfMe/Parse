# 文献综述：PARSE 奠基性研究分析

> 按论文引用顺序 [1]–[55]，逐篇分析结构化剪枝、知识编辑、机器遗忘、数据飞轮、Agent 系统、GRPO 强化学习与专用架构领域的关键文献。每篇综述涵盖：核心贡献、技术路线、与 PARSE 的关系。

---

## 一、数据飞轮与自改进训练 [1, 17–23]

### [1] AgenticQwen — 面向工业级工具使用的双数据飞轮小模型训练

Lyu 等人提出双飞轮架构训练小参数量的 Agent 模型。**合成飞轮**通过 Self-Instruct 扩展和 Persona 注入生成推理轨迹；**环境交互飞轮**生产 Agent-环境交互数据。两条飞轮协同，无需大规模教师蒸馏即可赋予小模型竞争力的工具调用能力。

**与 PARSE 的关系**：双飞轮机制被直接引入 PARSE 的第四阶段（康复），在 FFN 移植后为每个 (语种 × 学科 × 场景) 组合生成定向标定数据。

### [17] NVIDIA — 数据飞轮基础理论

NVIDIA 将数据飞轮定义为正向循环：模型产出改善训练数据，优质数据训练出更好模型，更好模型产出更优输出。这一递归自改进循环是现代小模型训练管线的理论基础。

### [18] ArenaLearning — AI 驱动的模拟竞技场

Luo 等人首创 AI 模拟竞技场，使模型在对抗环境中自动评估并生成训练数据。模型进行 pairwise 对阵，由 LLM 评判器打分，无需人工标注即可大规模生产偏好数据。

**与 PARSE 的关系**：ArenaLearning 的自动评估策略启发了 PARSE 自精炼飞轮中的 Critic 评分设计。

### [19] SRDF — 自精炼数据飞轮

Wang 等人引入 Generator-Navigator 协同对，通过迭代反馈环路不断生成更高质量的训练轨迹。Generator 提出序列，Navigator 评估并精炼。

### [20] IFDecorator — 可验证奖励的指令遵循强化学习

Guo 等人将指令遵循训练包装为可验证奖励驱动的 RL 优化，使奖励函数建立在可测量的任务完成度之上，而非学习的偏好模型。

### [21] UI-TARS-2 — 多轮 RL 的 GUI Agent

Wang 等人通过多轮强化学习推进 GUI Agent 训练，证明 Agent 能力从迭代环境交互中获益远超静态有监督数据集。

### [22] GAIA — GUI 测试时 Scaling Critic 系统

Wang 等人构建了训练 GUI Critic 模型的数据飞轮系统。Critic 评估生成的 GUI 交互轨迹并过滤低质量样本，形成自改进的评估-训练闭环。

**与 PARSE 的关系**：GAIA 的 Critic 模型设计直接指导了 PARSE 自精炼飞轮的质量评分机制。

### [23] SynthAgent / DeepSeekMath — 合成环境训练与 GRPO 算法

Lyu 等人提出在模拟世界中训练 Agent 语言模型的方法。该工作同时引入了 DeepSeekMath 的 GRPO 算法（详见第五部分）。核心洞见：程序化生成的环境可以无限量地产出多样化的 Agent 技能训练数据。

## 二、专用高效架构 [2, 54, 55]

### [2] MiniMind-O — 0.1B 参数的三模态 Omni 模型

Gong 提出 Thinker-Talker 双路径解耦架构。**Thinker** 路径处理语义理解（文本、视觉）；**Talker** 路径处理语音生成。表征通过中层桥接而非嵌入层或最终层传递。多 Token 预测（MTP）同时生成 8 层 Mimi 音频编码，仅 0.1B 可训练参数实现三模态（文本-音频-图像）处理。

**与 PARSE 的关系**：Thinker-Talker 的桥接层概念启发了动态能力路由器——来自语种、学科、场景三轴的表征通过调制门控汇聚，而非刚性架构路径。极致的参数效率证明架构创新可以弥补模型尺寸缩减。

### [54] Language Model Unlearning

该工作将 LLM 遗忘形式化为分布偏移问题，为定向移除知识提供了理论基础。PARSE 在确定哪些层可被安全替换时利用了这一原则：如果某层中的知识与事实可以被移除而不影响其他能力，那么该层本身也可以被替换。

### [55] Needle — 面向函数调用的纯注意力网络

Ndubuaku 等人提出了一个极端的架构主张：对于函数调用等结构化任务，FFN（约占标准 Transformer 65% 参数）完全冗余。注意力中的 softmax 操作已提供足够的非线性进行信息路由。最终的架构（纯注意力 + 门控残差 + ZCRMSNorm）以 26M 参数击败 270M-600M 通用模型。

**与 PARSE 的关系**：这是 PARSE FFN 移植策略的奠基性参考。Needle 证明 FFN 并非普遍必需；PARSE 将这一洞见从"全或无"扩展为**能力依赖**——推理关键层保留 FFN，对保留剖面无用的层才替换为 Needle 式纯注意力模块。

## 三、结构化剪枝 [3–16]

### [3] LLM-Pruner — 基于梯度耦合的结构剪枝

Ma 等人首创大语言模型的任务无关结构化剪枝。核心技术是梯度耦合分析：梯度高度相关的参数组被识别为功能单元，同保同剪。这保护了注意力机制（K、V 耦联）的内部一致性。

**与 PARSE 的关系**：LLM-Pruner 引入了参数组成功能单元的理念。PARSE 将这一理念从参数耦合扩展到**能力耦合**——层按其支持的能力形成功能组，而非仅靠梯度相关性。

**局限性**：全局稀疏度优化，无能力区分机制。所有任务共享同一个剪枝掩码。

### [4] SparseGPT — 一次性二阶剪枝

Frantar 和 Alistarh 通过高效的二阶 Hessian 近似实现 50% 非结构化稀疏。核心洞见：剪枝引起的误差可通过逆 Hessian 更新剩余权重来补偿，且可通过递归逐行计算避免显式矩阵构建。

**与 PARSE 的关系**：SparseGPT 证明重要性感知的权重补偿在大规模上可行。PARSE 将这一原则适配到三维重要性空间：不再*全局*补偿，而是仅针对用户要保留的能力进行补偿。

**局限性**：一次性剪枝缺乏恢复机制。如果初始剪枝决策对某个特定能力维度错误，无法纠正。

### [5] Wanda — 权值-激活乘积剪枝

Sun 等人将幅度剪枝简化为权重幅度与输入激活范数的乘积。仅需一次前向传播即可获得匹配 SparseGPT 的性能。优雅之处在于同时捕获静态权重重要性和动态输入敏感度。

**与 PARSE 的关系**：Wanda 的激活依赖性重要性评分在 PARSE 的 CIT 计算中被扩展。Wanda 用*输入*激活（每样本不同），PARSE 用*能力特定*的激活和梯度计算重要性，得分随 (语种 × 学科 × 场景) 组合而变化。

**局限性**：虽感知激活，但仍能力无关。无法区分中文语法驱动的激活模式与数学推理驱动的激活模式——两者都只是"激活"。

### [6] LaCo — 层级坍缩剪枝

Yang 等人从表征稳定性的视角审视剪枝。当连续层输出高度相似时（通过余弦相似度或 BERTScore 度量），中间层可以被坍缩而不损失信息流。将剪枝决策从"哪些参数小"转变为"哪些层相对邻居是冗余的"。

**与 PARSE 的关系**：LaCo 的表征中心视角与 PARSE 的 CIT 计算一致——两者都分析层*输出*而非层*权重*。PARSE 进一步追问：对于*哪个能力*而言表征相似？两层在翻译任务上的输出可能完全一致，但在逻辑推理上分歧显著。

### [7] ShortGPT — 层冗余性实证发现

Men 等人提供了 LLM 层大规模冗余的确凿实证证据。除证明 25% 以上的层去除后对平均困惑度影响极小外，ShortGPT 揭示了一个关键发现：层移除对不同能力的冲击**极不均匀**。深层对推理贡献呈指数级攀升，浅层主要处理语法。

**与 PARSE 的关系**：这是 PARSE 整体方法的实证基石。ShortGPT 证明"重要性"没有唯一答案——层的"重要性"完全取决于你测量的是哪种能力。PARSE 将这一观察系统化为三维 CIT。

### [8] LayerDrop — 训练期结构化 Dropout

Fan 等人在训练中引入随机丢弃整层的结构化 Dropout。训练出的模型在推理时可灵活移除任意层，并支持按输入动态选择深度。

**与 PARSE 的关系**：LayerDrop 的核心理念——并非所有层对所有输入都是必需的——在 PARSE 中被从*输入复杂度*扩展到*能力特异性*。中文输入和数学问题即使 token 数相同，也可能需要不同的层。

### [9] DeeBERT — 基于置信度的早退机制

Xin 等人将轻量分类器附加到 BERT 中间层。当分类器置信度超过阈值时提前退出，绕过后面的深层。简单样本用浅层，困难样本用全深度。

**与 PARSE 的关系**：DeeBERT 证明推理时的层选择是可行的。PARSE 将这一概念从样本难度推广到能力类型，使用 DCR 基于检测到的任务语境动态选择有效深度。

### [10] FastBERT — 自蒸馏自适应推理

Liu 等人将早退机制与自蒸馏结合：全深度模型（教师）将知识蒸馏到浅层出口（学生），确保早退预测质量且无需单独训练。

### [11] oBERT — 面向 BERT 的最优二阶剪枝

Kurtic 等人通过迭代幅度剪枝加权重倒回实现 BERT 类模型的行业领先压缩比。"最优"指理论上最小化损失局部二次近似的二阶剪枝标准。

### [12] Movement Pruning — 通过微调学习自适应稀疏性

Sanh 等人提出在微调中学习剪枝目标。利用一阶梯度信息判定参数重要性——不同于幅度剪枝（静态），Movement Pruning 的稀疏模式随微调目标变化，是任务感知的。

**与 PARSE 的关系**：Movement Pruning 是能力感知剪枝的概念前身。通过使稀疏性依赖于任务，它证明"更少"可以意味着"对特定目标更好"。PARSE 将其从单任务感知推广到多维能力感知。

### [13] BERT-of-Theseus — 渐进式模块替换

Xu 等人将压缩建模为渐进模块替换。较小的替换模块逐步替换原始模块。核心洞见：训练稳定性至关重要——一次性激进压缩导致灾难性遗忘，渐进替换允许模型适应。

**与 PARSE 的关系**：PARSE 的移植策略遵循同一原则。FFN 模块被*替换*而非简单删除，DCR 门控扰动初始化为零以确保模型从稳定状态开始适应。

### [14] TinyLlama — 小模型大规模预训练

Zhang 等人证明以 1 万亿 token 预训练 1.1B 模型可以取得竞争力性能。这建立了小架构并非先天受限——它们只是需要足够训练数据的实证基础。

**与 PARSE 的关系**：TinyLlama 提供了能力上界参考：这是训练充分的小模型能达到的水准。PARSE 的目标是通过智能压缩大预训练模型接近这一上界，而非从头训练。

### [15] MInference — 长上下文动态稀疏注意力

Jiang 等人通过动态稀疏注意力模式加速长上下文 LLM 的预填充阶段。识别并利用注意力矩阵中的稀疏性，降低长序列的二次复杂度瓶颈。

### [16] LLM-Shearing — 结构化剪枝 + 持续预训练

Xia 等人提出灵活的结构化剪枝后接持续预训练进行能力恢复。两阶段方法解耦了剪枝决策（删除什么）与恢复过程（如何补偿），允许更激进的初始剪枝。

**与 PARSE 的关系**：剪枝+恢复范式直接对应 PARSE 的第三阶段（移植）+ 第四阶段（康复）管线。PARSE 将通用的持续预训练替换为定向的能力特定飞轮恢复。

## 四、Agent 系统与工具调用 [24–30]

### [24] Gorilla — 连接海量 API 的 LLM

Patil 等人首创基于 LLM 的 API 调用，通过在大量 API 文档语料上微调 LLaMA 模型实现。推理时的检索增强为模型提供最新的 API 签名。

**与 PARSE 的关系**：Gorilla 证明工具使用是可训练技能而非规模涌现属性——这是 PARSE 声称函数调用能力可在小型压缩模型中保留的关键前提。

### [25] xLAM — 大规模动作模型家族

Zhang 等人将 API 调用范式扩展为 1B 至 8×22B 的模型家族，在 Berkeley Function-Calling Leaderboard 上全面占据领先地位。

### [26] TinyAgent — 边缘设备函数调用

Erdogan 等人将 1.1B 和 7B 函数调用模型部署到消费级 MacBook 上，在函数调用基准上超越 GPT-4-Turbo。这直接验证了 PARSE 的目标部署场景。

### [27] ToolFlow — 基于图采样的工具组合

Wang 等人仅使用 8,000 条高质量对话样本，在 LLaMA-3.1-8B 上达到 GPT-4 级工具调用。证明样本效率——而非数据量——是工具使用能力的瓶颈。

**与 PARSE 的关系**：ToolFlow 的样本效率理念与 PARSE 的 CIT 方法一脉相承：认识*哪些样本*对*哪种能力*重要，比在海量未过滤数据集上训练更有价值。

### [28] SLM Agent 系统综述

Sharma 和 Mehta 提供了首个小语言模型 Agent 架构、能力和训练方法的系统综述，建立评估基线并识别人能力差距。

### [29] TinyLLM — 边缘设备 Agent 评估

Haque 等人对部署在边缘设备上的小语言模型进行 Agent 任务基准测试，提供超越抽象准确率分数的硬件感知评估指标。

### [30] CAMPHOR — 协作式多 Agent 架构

Fu 等人提出面向端侧部署的层次化多 Agent 架构。推理 Agent 分解复杂任务，专家 Agent 执行子任务，协调者管理 Agent 间通信。

**与 PARSE 的关系**：CAMPHOR 的层次化分解镜像了 PARSE 的三维能力分解：复杂能力分解为可独立处理的正交维度。

## 五、GRPO 强化学习 [23, 31–36]

### [23] DeepSeekMath / SynthAgent — GRPO 算法

DeepSeekMath 团队提出 Group Relative Policy Optimization。不对每个提示训练独立的价值网络（Critic），而是采样多个输出并通过对组内相对比较估计优势函数。这完全消除了 Critic 模型，将 RL 训练管线从四个模型降至三个。

**与 PARSE 的关系**：GRPO 的样本效率和无 Critic 设计使其成为 PARSE 移植后康复阶段的理想选择，该阶段计算预算有限且需要能力特定的奖励函数。

### [31] SLM-ToolUse-GRPO — 面向工具调用的 GRPO

Paprunia 等人专门研究 GRPO 增强小语言模型工具使用能力。奖励函数针对 JSON 结构有效性、工具选择准确性和参数精度这三个函数调用质量的正交维度。

### [32] EBPO — 经验贝叶斯收缩稳定 GRPO

Han 等人通过经验贝叶斯收缩估计器解决 GRPO 在小批量设置中的估计方差问题。收缩将嘈杂的组内基线向全局先验正则化，在组规模有限时保持训练稳定性。

### [33] STAPO — 沉默伪 Token 以稳定 GRPO 训练

Liu 等人发现"伪 token"现象：约 0.01% 的 token 对目标函数贡献可忽略，却获得不成比例的大梯度更新，导致训练不稳定。解决方案是在梯度计算中沉默这些 token。

**与 PARSE 的关系**：PARSE 的移植涉及大量门控值接近零的层——恰好是伪 token 可能扰乱训练的条件。STAPO 的沉默机制提供了直接的技术解决方案。

### [34] Mu-GRPO — 离策略效率

Tian 等人证明 GRPO 能容忍远超预期的 rollout 延迟。通过解耦 rollout 生成与模型更新，达到约 2× 训练加速而不牺牲质量。

### [35] ActFocus — 解决动作瓶颈

He 等人发现 Agent 轨迹中梯度信号集中在少量动作 token（例如工具调用参数）上。Token 级能量重加权将学习信号重新分布到完整轨迹上，多环境下性能提升超 60 个百分点。

### [36] ChemCRAFT — 专业领域的 Agent RL

Li 等人将 Agent 强化学习应用于面向药物设计的化学语言模型。领域特定的奖励工程使小模型在专业科学任务上超越云端大模型。

## 六、知识编辑 [37–41]

### [37] ROME — 秩-1 模型编辑

Meng 等人在 GPT 类模型中首次实现了通过因果追踪定位事实知识。通过分析每层激活在事实检索过程中的贡献，他们精确定位到存储事实关联的具体 MLP 层。编辑通过闭式秩-1 权重更新完成，修改目标事实行为的同时保持无关知识不变。

**与 PARSE 的关系**：ROME 是 PARSE 核心前提的奠基参考——LLM 知识可在具体层被定位。如果一个事实可被追踪到 Layer 18 的 MLP 激活 245-267，那么整个能力（如数学推理）同样可被追溯到一组层。这是知识编辑与能力保持型压缩之间的理论桥梁。

### [38] MEMIT — Transformer 中的批量记忆编辑

Meng 等人将 ROME 扩展为批量编辑。通过约束优化在一次更新中修改数千条事实，同时防止编辑间相互干扰。核心技术贡献是在保持特异性的前提下解决"同时大量修改"问题。

**与 PARSE 的关系**：MEMIT 证明多个知识修改可以无交叉干扰共存——直接类比 PARSE 的多能力保持：保护中文语法不可干扰保护英文数学。

### [39] MEND — 基于梯度分解的模型编辑网络

Mitchell 等人训练超网络预测参数更新以实现高效编辑。通过将梯度分解为低秩分量，MEND 学会从编辑请求本身生成最优权重扰动，将计算跨编辑分摊。

### [40] SERAC — 基于外部记忆的大规模模型编辑

Mitchell 等人提出了权重直接修改的替代方案：范围分类器将查询路由到基座模型或反事实记忆缓存。这完全将编辑与模型参数解耦。

**与 PARSE 的关系**：SERAC 对"模型知道什么"与"我们想让它输出什么"的分离，平行于 PARSE 对"模型有什么能力"与"我们想保留什么能力"的分离。其路由机制是 DCR 的概念前身。

### [41] 知识编辑综述

Wang 等人提供了知识编辑方法的系统化分类法，按修改目标（权重 vs. 外部记忆）、编辑范围（单条 vs. 批量）和保留保证分类。

## 七、机器遗忘 [42–47]

### [42] SISA — 分片、隔离、切片、聚合

Bourtoule 等人提出了基础的遗忘框架。训练数据被分割为分片，独立子模型在各自分片上训练。遗忘请求仅需重新训练受影响分片的子模型。这同时提供了效率和可审计性。

**与 PARSE 的关系**：SISA 的分片原则证明局部更改（一个分片）不必影响全局性能（其他分片）——直接类比 PARSE 的主张：移除某一能力的层不必影响其他能力。

### [43] 机器遗忘综述

Yao 等人提供了跨领域的遗忘方法全面综述，建立评估指标并按合规保证级别（精确、近似、概率）对方法分类。

### [44] LLM 的知识遗忘

Liu 等人聚焦 LLM 背景下的遗忘问题，处理无界输出空间和定义生成模型"遗忘"的独特挑战。

### [45] 灾难性遗忘

French 关于神经网络中灾难性遗忘的奠基性工作。虽然主要是持续学习的挑战，但也为理解*为什么*激进剪枝会退化性能提供了洞见：移除参数会以不成比例地影响学得好但脆弱的表征的方式改变损失景观。

**与 PARSE 的关系**：理解灾难性遗忘机制对设计 PARSE 的康复阶段至关重要。移植必须通过定向能力特定微调补偿 FFN 移除引发的遗忘。

### [46] Descent-to-Delete — 基于梯度的遗忘

Sekhari 等人将遗忘形式化为对目标数据的梯度*上升*加 KL 散度约束。理论框架提供删除保证，实际实现能进行高效的近似遗忘。

### [47] Fast Machine Unlearning — 无需重新训练

Golatkar 等人通过分析参数对特定数据点的 Fisher 信息矩阵敏感性，解析地计算"最优遗忘方向"，近似遗忘更新而无需重新训练。

**与 PARSE 的关系**：基于 Fisher 的"哪些参数对哪些数据重要"的识别方法直接对应 PARSE 的 CIT 计算——利用激活和梯度双信号识别"哪些层对哪些能力重要"。

## 八、任务特定压缩与效率 [48–51]

### [48] LLM 的任务特定压缩

该工作提出压缩应以目标任务为引导，而非全局优化标准。任务相关的结构保留，任务无关的结构激进压缩。

**与 PARSE 的关系**：这是 PARSE 能力特定压缩的直接前身。PARSE 将概念从单任务特定扩展到多维（语种 × 学科 × 场景）能力感知。

### [49] 通过 Priming 和剪枝实现紧凑语言模型

该工作处理了特定语言理解任务需要多少架构容量的互补问题，提出在剪枝前通过 Priming 初始化。

### [50] BERT QA 模型的结构化剪枝

McCarley 等人提供了早期证据，证明针对特定下游任务（问答）的结构化剪枝可以实现大幅压缩而不引起灾难性精度损失，确立了任务特定剪枝范式。

### [51] 参数高效微调综述

Ding 等人全面综述了 PEFT 方法：适配器、前缀微调、LoRA 和提示微调。这些技术在微调中仅修改极小部分参数，实现高效适配。

**与 PARSE 的关系**：PARSE 的 DCR（0.08M 参数，模型大小的 0.09%）本质上是一个 PEFT 模块——一个微小的可训练组件，调制冻结的基座架构。DCR 之于能力路由，犹如 LoRA 之于任务适配。

## 九、训练稳定性与干预 [52, 53]

### [52] 推理时干预 — 引出真实回答

Li 等人证明在推理时激活特定的注意力头模式可以将模型行为转向真实性，无需任何权重修改。这建立了模型行为可通过激活空间干预来控制的实证基础。

**与 PARSE 的关系**：PARSE 的 DCR 门控调制——基于输入语境动态缩放移植模块的贡献——本质上是一种推理时干预。原则相同：通过激活调制而非权重修改控制行为。

### [53] 向良好校准的 LLM 正则化

该工作研究微调过程中保持校准的正则化，确保压缩或适配后的模型维持可靠的置信度估计，而不是在保留能力上过分自信、在其他能力上不足自信。
